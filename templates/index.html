Hướng dẫn: Dùng LivestreamVoice để đọc comment livestream & tích hợp OBS

Cập nhật: 24/06/2025 · Thời lượng đọc: 6 phút

Tóm tắt nhanh

LivestreamVoice cho phép đọc comment (TTS), auto-reply, và có chức năng Copy OBS Source để bạn dán trực tiếp vào OBS (Browser Source). Quy trình chuẩn: Chọn nền tảng → Nhập username/link → Start → Copy OBS Source → Dán vào OBS → Cấu hình audio & CSS nếu cần.

Hướng dẫn chi tiết (theo thứ tự)

  1. Truy cập & chọn nền tảng

    Mở livestreamvoice.com. Chọn TikTok (form username), hoặc chuyển sang phiên bản Facebook / YouTube nếu bạn stream trên các nền tảng đó.

  2. Nhập thông tin nền tảng
    • TikTok: nhập username (KHÔNG có @).
    • Facebook: dán link livestream (URL trực tiếp của live).
    • YouTube: dán link video hoặc livestream.
  3. Bật tính năng (TTS / auto-reply / cảm ơn)

    Sau khi nhập, chọn các tùy chọn bạn muốn: Đọc comment (TTS), AI auto-reply, Cảm ơn thả tim, Cảm ơn tặng quà. Nhấn Start để bắt đầu lấy comment và xử lý.

  4. Lấy OBS Browser Source

    Bấm Copy OBS Source trên trang — trang sẽ tạo link/embed động (JS client-side). Mở OBS → Sources → + → Browser → dán URL đó vào ô URL → chỉnh Width/Height → OK.

    Gợi ý kích thước & tuỳ chọn OBS
    • Ví dụ banner ngang: 1280x150. Hộp comment bên: 400x600.
    • Bật Refresh browser when scene becomes active nếu muốn reload tự động khi chuyển scene.
    • Muốn hiệu ứng/animation mượt: bật Shutdown source when not visible để tránh giữ state không cần thiết.
  5. Quản lý âm thanh (TTS)

    TTS phát từ trình duyệt. Để tách/kiểm soát âm thanh:

    • Nếu OBS hiển thị fader cho Browser Source, dùng fader đó để điều chỉnh volume.
    • Nếu không tách được, dùng virtual audio cable (VB-Cable trên Windows, BlackHole trên macOS) để route âm thanh từ trình duyệt sang input riêng cho OBS (thêm bằng Audio Output Capture).
    • Muốn mute/monitor riêng: dùng Advanced Audio Properties trong OBS để set monitoring / mute cho nguồn đó.
  6. Nền trong suốt & CSS cho overlay

    Nếu bạn muốn overlay chỉ hiển thị chữ (không có nền màu), có 2 cách:

    • Nếu URL OBS Source hỗ trợ parameter (ví dụ ?transparent=true), thêm vào URL.
    • Nếu không, dán URL vào Browser Source rồi copy đoạn Custom CSS sau vào ô Custom CSS của Browser Source:
    html, body { background: transparent !important; }
            body { margin: 0; font-family: Inter, system-ui, -apple-system, "Segoe UI", Roboto, "Helvetica Neue", Arial; }
            .comment-item { padding: 6px 10px; background: rgba(255,255,255,0.8); border-radius: 8px; margin-bottom:6px; }
            .comment-item .author { font-weight:700; font-size:0.95rem; }
            .comment-item .text { font-size:0.95rem; color:#0f172a; }
            @media (max-width:600px) {
              .comment-item { padding:8px; font-size:14px; }
            }

Lưu ý & Troubleshooting

OBS Source là link động: nút Copy OBS Source sinh link bằng JavaScript trên client. Bạn cần bấm nút trên trang để lấy link — mình không thể tạo link thay bạn.

Audio bị trộn: nếu TTS bị trộn vào Desktop Audio, ưu tiên tách bằng virtual audio cable để có control riêng trên OBS.

CORS / quyền API: một số tính năng (auto-reply, kết nối API) có thể cần quyền/headers hợp lệ. Mở DevTools (F12) nếu chức năng không chạy như mong muốn để xem lỗi console/network.

Giao diện overlay khác web: do CSS/JS của site. Trong trường hợp đó, bạn có thể dán OBS Source URL cho mình — mình sẽ viết CSS tùy chỉnh để ẩn các thành phần thừa, tối ưu font/kích thước và làm transparent đúng cách.

Hướng dẫn: Thiết lập OBS để hiển thị comment lên màn hình từ LivestreamVoice

1. Tạo Browser Source trong OBS

Mở OBS → Sources → + → Browser. Đặt tên ví dụ "LivestreamVoice Comments". Trong URL nhập đường dẫn đã copy khi nhấn nút Copy OBS source (ví dụ https://livestreamvoice.com/tiktok/widget/bongda24h). Điều chỉnh Width/Height (ví dụ: 800x200) và chọn 'Refresh browser when scene becomes active'.

2. Chỉnh CSS hiển thị

Trong Browser Source, bạn có thể dùng CSS để ẩn nền, thay font, thay màu. Để nền trong suốt, bật 'Custom CSS' hoặc dùng parameter transparent=true nếu backend của bạn hỗ trợ.

3. Tối ưu hiển thị comment

Gợi ý: giới hạn số comment đang hiển thị (5-10), thiết lập animation fade để không che khung, đặt tốc độ cuộn phù hợp với lượng comment.

4. Kiểm tra đường truyền & quyền

Đảm bảo API lấy comment không bị CORS chặn (server trả header Access-Control-Allow-Origin: * hoặc domain của bạn). Nếu gặp lỗi CORS, log console trong trình duyệt sẽ cho biết — sửa tại backend.
